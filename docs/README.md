# Documentation Index

This folder keeps planning and execution artifacts organized for future prompts.

## Structure

- `docs/plans/`: implementation and improvement plans
- `docs/tasks/`: task boards and to-do lists
- `docs/kanban/`: kanban tracking files
- `docs/reports/`: QA, validation, and implementation reports
- `docs/audits/`: bug audits and technical audits
- `docs/summary/`: project summary/context for next prompting
- `docs/competition/`: demo and competition support docs

## Key Entry Points

- Project summary: `docs/summary/PROJECT_SUMMARY.md`
- DB fix plan/tasks/kanban:
  - `docs/plans/DATABASE_FIX_PLAN.md`
  - `docs/tasks/DATABASE_FIX_TASKS.md`
  - `docs/kanban/DATABASE_FIX_KANBAN.md`
- UI/UX plan/tasks/kanban:
  - `docs/plans/UI_UX_PLAN.md`
  - `docs/tasks/UI_UX_TASKS.md`
  - `docs/kanban/UI_UX_KANBAN.md`
- Community plan/tasks/kanban:
  - `docs/plans/COMMUNITY_PAGE_IMPROVEMENT_PLAN.md`
  - `docs/tasks/COMMUNITY_TASKS.md`
  - `docs/kanban/COMMUNITY_KANBAN.md`
